Monday, June 11: XPath

Introduction to XPath in eXist-db (10:00 a.m.–12:00 p.m.)

  • Simple XPath expressions (60 minutes)
    1. Open http://newtfire.org:8338/exist/apps/eXide/index.html, click “New XQuery”, and erase all content in the editing window. You’ll type your XPath in the editing window and run it with the “Eval” button.
    2. All XPath expressions return a sequence. Sequences may contain nodes (elements, attributes, etc.) or atomic values (strings, numbers, etc.) A sequence of one item is nonetheless a sequence, as is an empty sequence.
    3. Strings and numbers (atomic values) are XPath expressions
      • "Hi, Mom!" (strings are enclosed in single or double quotation marks)
      • 1 (numbers are not enclosed in quotation marks)
      • 1.0 (What should this return?)
    4. Arithmetic expressions are XPath expressions
      • 1 + 1
    5. XPath library functions (with no arguments) are XPath expressions
      • current-date()⬀
      • current-dateTime()⬀
    6. XPath library functions (with arguments) are XPath expressions
      • upper-case('dhsi')⬀
      • count(('Curly', 'Larry', 'Moe'))⬀ (Why two sets of parentheses?)
    7. Nested XPath library functions and operations are XPath expressions
      • max((1 + 2, 10 div 5, 6 * 0.2)) (Remember those two sets of parentheses?)
      • translate(upper-case('Hi, Mom!'),'AEIOU','xxxxx')
      • format-dateTime(current-dateTime(),'[h].[m01][Pn] on [FNn], [D1o] [MNn]')⬀
    8. Nested functions are hard to read. Use the arrow operator (=>) instead
      • upper-case('Hi, Mom!') => translate('AEIOU','xxxxx')
      • current-dateTime() => format-dateTime('[h].[m01][Pn] on [FNn], [D1o] [MNn]')
  • XPath axes and context nodes (30 minutes)
  • XPath path steps (30 minutes)
